# ✅ Чеклист перед развертыванием

## Перед загрузкой на сервер:

- [ ] Запустите `python scripts/cleanup.py` для очистки временных файлов
- [ ] Убедитесь что файл `files/Шаблон.docx` присутствует в репозитории
- [ ] Проверьте что все зависимости в `requirements.txt` актуальны
- [ ] Убедитесь что `.gitignore` настроен правильно
- [ ] Проверьте что переменные окружения НЕ закоммичены (нет `.env` файла)

## Файлы которые ДОЛЖНЫ быть в репозитории:

### Обязательные:
- [x] `api.py` - веб-сервер
- [x] `run_all.py` - запуск всех сервисов
- [x] `wpd/` - вся папка с модулями ядра
- [x] `tgbot/bot.py` - Telegram бот
- [x] `files/Шаблон.docx` - **КРИТИЧЕСКИ ВАЖНО!**
- [x] `requirements.txt` - зависимости
- [x] `Procfile` - для Railway/Heroku
- [x] `railway.json` - конфигурация Railway
- [x] `config/Dockerfile` - для Docker (опционально)

### Документация:
- [x] `README.md`
- [x] `docs/DEPLOY.md`
- [x] `docs/QUICK_START.md`

## Файлы которые НЕ должны быть в репозитории:

- [ ] `venv/` - виртуальное окружение
- [ ] `__pycache__/` - кэш Python
- [ ] `*.pyc` - скомпилированные файлы
- [ ] `result.docx` - временные результаты
- [ ] `files/results/*` - результаты обработки
- [ ] `files/uploads/*` - загруженные файлы
- [ ] `.env` - переменные окружения (настраиваются на сервере)

## Настройка на сервере:

- [ ] Установлены переменные окружения:
  - `PPLX_API_KEY`
  - `TELEGRAM_BOT_TOKEN` (если используется бот)
- [ ] Установлены все зависимости из `requirements.txt`
- [ ] Проверена работоспособность веб-сервера
- [ ] Проверена работоспособность Telegram бота (если используется)

## После развертывания:

- [ ] Веб-интерфейс доступен по URL
- [ ] Можно загрузить файл и получить результат
- [ ] Telegram бот отвечает на команды (если настроен)
- [ ] Логи не содержат критических ошибок
